# Phase 2 Supplier Bills

## Scope Implemented

- Supplier bills with bill type, tax, total, outstanding amount, payment status, remarks, and supporting-document status.
- Recurring obligations with monthly draft generation and duplicate prevention by database uniqueness.
- Automatic missing-draft check when the dashboard opens, plus manual generation from `/recurring`.
- Payment vouchers with multi-item-ready schema and atomic monthly numbering through the database.
- Bill payments for partial/multiple payments and later bank matching.
- Private document metadata, document links, archive/restore, replacement metadata, duplicate hash warning, and upload audit logging.
- Private Supabase Storage bucket `bill-documents`.
- Missing-document tracking page.

## Required Database Migration

Apply:

```text
supabase/migrations/0004_phase2_supplier_bills.sql
```

This migration is additive. It creates new Phase 2 tables, private storage bucket metadata, RLS policies, storage policies, and helper functions.

## Document Upload Rules

- Allowed: PDF, JPG, JPEG, PNG, phone photos, screenshots.
- Max size: 10 MB per file.
- Files are uploaded through `/api/documents/upload`.
- MIME type is validated server-side.
- Stored filename is generated by UUID.
- Original filename is retained as metadata.
- Documents use signed URLs only; no public URLs.

## Payment Voucher Numbering

Format:

```text
{ENTITY_CODE}/PV{YY}-{MM}/{SEQUENCE}
```

Examples:

- `IEA/PV26-07/001`
- `IETA/PV26-07/001`
- `PLC/PV26-07/001`
- `KALER/PV26-07/001`

The sequence is per entity/month and generated by `public.generate_payment_voucher_number`.

## Rollback

- App rollback: revert the Phase 2 deployment.
- Database rollback: leave additive tables in place and hide Phase 2 routes.
- Do not drop tables, delete documents, or remove the bucket unless production data is reviewed and deletion is approved.

## Acceptance Checklist

- Owner can create supplier bill.
- Staff sees only entity-based bill/PV records permitted by `user_entity_access`.
- Authorised staff can view documents across entities.
- Read-only users can view/download documents but cannot upload/manage.
- Upload PDF/image under 10 MB succeeds.
- Oversized or unsupported file is rejected.
- Multiple upload and phone camera input are available.
- Download uses signed URL.
- Archive/restore records audit events.
- Duplicate hash warning appears where practical.
- Missing-document report lists incomplete records.
- Recurring generation creates one monthly draft only.
- Recurring generation can issue PV.
- Three-day due reminder list appears.
- PV numbering follows entity/month sequence and uniqueness.
- Cancelled numbers remain used by database uniqueness/audit trail.
- Staff cannot access owner-only bank balances unless separately granted.
